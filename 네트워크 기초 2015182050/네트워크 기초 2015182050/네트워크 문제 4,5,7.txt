답 : repeater - 증폭기


2. bridge 가 하는 일과 존재이유에 대해 서술하시오.

답 : bridge는 LAN을 작은 단위(segment)로 나눠서 설치할때 segment 사이를 연결할 때 사용한다.
     같은 segment 사이에서 신호를 보낼 때는 브릿지가 막아버리고 다른 segment로 신호를 보낼때는 통과시킨다.
     브릿지가 osi 2계층의 Mac address를 이용해서 segment가 다른지를 구분하는 작업을 한다.
     브릿지 내에 Mac address 를 저장하는 테이블을 만들고 데이터가 이동할때 이 테이블을 확인해서 넘길지 말지 결정     

    
     브릿지의 존재이유 : 쓸데없는 트래픽을 줄여주고, 전체 성능이 올라간다. 



3. bridge 와 switch 의 차이점은?

답 : 브릿지는 한번에 하나의 데이터만 처리 가능하지만, 스위치는 여러개의 데이터를 동시에 처리 가능하다

      
4. 부가적인 기능을 가진 스위치 중 store and forward switch 와 cut through switch 를 설명하시오.

답 : store and forward 스위치는 이동중인 데이터를 모두 저장하고 문제가 없으면 보내고, 문제가 있으면 삭제하는 스위치이다.

     신뢰성은 좋지만 모슨 데이터를 검사하기 때문에 시간이 오래걸린다.

     cut through 스위치는 데이터를 오류 체크 없이 모두 보낸다. 속도는 높지만, 신뢰성은 떨어진다.

     서비스의 용도에 따라 어떤 것을 쓸지 달라짐

     은행 -- 신뢰성이 중요 위에꺼사용 , 
     iptv -- 동영상 정보를 주고받는게 중요, 점하나 깨지고 이런건 중요하지않음, 신뢰성보다는 속도 아래꺼 사용




5. 라우팅이 하는 일과 두가지 라우팅 방식에 대해 서술하시오.

라우팅은 네트워크와 네크워크 사이에서 데이터가 목적지 까지 가장 빠르게 갈 수 있는 경로를 찾아주는 장치이다.

IP address를 사용해서 경로를 찾고, broadcast 메세지를 모두 걸러버린다.

라우팅 방식으로는 

dynamic routing -- 매번 다양한 길로 보낼 수 있고 전체 네트워크 경로를 골고루 사용할 수 있다
                   단점은 매번 보낼떄마다 계산을 해야함, 시간이 걸림


static routing  -- 경로를 고정해버리는 방식, 길계산을 할 필요 없음, 계산 과정이 없어서 속도 좋음
                   문제는 그 경로에 트래픽이 몰리면 전송 지연이 생길 수 있음.
                   일반적으로 쓰이지는 않고 전체적인 트래픽을 지정 할 수 있을때 사용
                   이미 네트워크가 다 구축 되었을때 사용

21. CDMA 가 TDMA 보다 좋은점에 대해서 서술하시오.


   1. 보안에 강하다
     TDMA는 시간을 쪼개서 여러사람의 정보를 보내는 방식
     CDMA는 암호화된 코드로 전달, 그 안의 헤더를 해석 할 수 있어야 해독가능

   2. 하나의 셀에서 서비스할 수 있는 단말기의 갯수가 많음
     CDMA가 셀타워를 좀더 적은 수로 세워도 가능하다.
     즉, 시설 비용을 좀 줄일 수 있음

   3. hand-off를 좀더 줄 일 수 있음
      핸드오프를 할 때 많은 정보를 주고 받아야함  (베터리 소모가 큼)
      CDMA는 이 과정이 좀더 적음 (베터리 소모가 더 적음)
