cp3 topology, architecture


topology 랜을 구성하고 전송매체를 깔때 어떤모양으로 깔거냐? 라는것을 기준으로 나누는것

논리 topology
데이터의 이동 경로를 보고 정의한 개념

물리 topology
생긴것만 가지고 정의한 개념

bus topology

하나가 보내주면 나머지가 다받음 (막아줄 장치가 없기 때문에)

1 : all

동축케이블로 구성

물리적으로나 논리적으로나 bus임

장점 : 구현하기가 쉽다

단점 : 어느 한군데가 고장나면 원인파악하기가 쉽지가 않음

       한군데가 손상되면 전체가 다운되버림  fail

       하나가 보내면 나머지가 다 받아야 하는데 다 못받을 수 있게됨




unicast   : 1대1 통신

broadcast : 1 대 all(전부)

multicast : 1 대 n (전부가 아닌) 









star topology 

가운데 연결장치가 Hub인 경우가 있고, switch인 경우가 있음

hub (신호를 받아서 연결되 있는 모든 애들에게 다 뿌려줌, 어디서왔는지 이런건 관심 없음)

물리적으로는 star 

논리적으로는 bus



switch 는 데이터가 어디서 왔는지 어디서 가는지를 구분해 낼 수 있음 즉, 목적지에다만 데이터 보내는게 가능

물리 논리적으로 다 star



장점 : 구현이 쉽다, 케이블 하나 망가진다고 전체가 망가지지는 않음

단점 : 중간 연결장치가 고장나면 전체적으로 문제가 생김








ring topology

건물들을 쭉 이어서 할때 ring 형태 많이 이용

선을 두개를 깜 dual ring 이유는?

1.하나는 시계방향 하나는 반시계 

2.하나는 백업용

장점 : 직접 연결 안하고 전체를 연결할수 있음 (외곽 순환도로)

단점 : 링한군데가 문제 생기면 전체 네트워크에 문제 생김
 -> dual ring 으로 극복







wireless topology


access point를 이용해서 구축 cell

물리적으로 생긴형태는 star

논리적으로는 bus형태 (만약 1대1 통신만 지원한다 하면 star)


장점 : 설치가 쉽다 (선을 깔 필요가없음), 네트워크 장비들을 cell과 cell사이를 돌아다니면서 사용가능

단점 : access 포인트 근처가 아니면 신호가 약해짐, 보안에 취약함
