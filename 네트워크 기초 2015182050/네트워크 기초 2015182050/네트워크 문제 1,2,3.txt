3.랜이 대부분 distributed processing을 쓰는 반면에, 현재 모바일은 mainframe 방식을 선호한다. 그 이유는?


 이유 : 모바일 디바이스의 인기 올라감

 휴대성이 좋아야함

 배터리 소모가 적어야함

 따라서, 모바일 기기가 하는일이 많으면 배터리도 커야하고 무게도 많이나감

 그래서 클라이언트에선 데이터만 보내고 서버가 연산 다해주고 처리 결과만 보냄

 Thin Client -----네트워크 속도가 올라가서 가능------> server   (서버가 다해주니까 기기는 가벼워짐)



4. 네트워크를 이용하는 이유는?? (과거, 현재)

과거에는 리소스를 나눠쓰기위해서 네트워크를 사용했음

컴퓨터보다 프린터가 비싸던 때, 프린트를 연결해서 같이쓰자는 의미로 네트워크

요즘은 보안 문제 때문에 네트워킹을 많이 씀

네트워크에 출력기록이 모두 나옴 누가 몇시에 출력했는지

그 문서가 회사 밖에서 발견되면 누가 유출했는지 바로 알 수 있음.

10. hub와 switch의 차이는?

답: hub는 신호를 받아서 연결되있는 모든 노드에 신호를 보냄 (broadcast), hub로 star topology를 구현하면 물리적으로는 star, 논리적으로는 bus가 됨


11. ring topology의 장단점과 단점의 해결방안은?

답 : 장점 직접 연결을 하지 않고도 전체를 연결할 수 있음

     단점 : 한군데가 끊어지면 전체 네트워크에 문제가 생김   ---> 듀얼링으로 극복



12. topology 와 architecture의 차이는??

답 : topology는 데이터가 어떤 모양으로 이동하는지 정의 (외향적인것)
     
     architecture는 데이터를 어떤 방식으로 이동시킬껀지 정의 (좀더 세분화)



13. 이더넷의 access method에 대해 서술하시오.


답 : 

carrier sense : 이더넷으로 구축된망에 데이터를 이동시키전에 누군가 데이터를 보내고 있는지 확인하는것

                 전기적인 신호로 보내기때문에 전류가 감지되면 케이블에 데이터가 이동중, 데이터 보내지 않음
                  
                전기적 신호가 있는지 감지

multiple access  : 모든 네트워크 장치들이 전송매체에 동등한 접근을 허용하는 것



collision detection : 충돌 파악, 데이터가 간발의차이로 동시에 올라갓을때 충돌을 감지하고 랜덤만큼의 
 
                      기다리게 한후 다시 보내줌


CSMA/CD  의 장점 : 가격이쌈, 많이 쓰임


             단점 : 충돌이 많이 발생할때는 효율이 좋지않음
                     (이더넷에 많이 연결될수록 충돌이 더많이 일어남)
